How The Unit testing Framework Works


We've provided a large number of Unit testing functions that you can use in 2048. Your main task will be filling in all of the problems in the spec; however, using the functions that we have provided to you will make writing the required codes much easier. In addition, in this assignment we're going to use a new framework called Unit testing to verify the correctness of the codes that we write.

To understand how the Unit testing framework works, let's start by looking at its structure.


import unittest

class TestGame(unittest.TestCase):

    def test_action(self):
        self.assertEqual(FOO, expected_result)

    def test_another_action(self):
        self.assertTrue(FOO)
        self.assertFalse(FOO)

    def test_third_action(self):
        with self.assertRaises(TypeError):
            function(something_erroneous)

if __name__ == '__main__':
    unittest.main()

Based on the structure of the function, we notice that the class for your unittest, functions within the class, intended input, and expected output. This is very similar to the other HOFs we have already seen! Let's take a look at some examples.

  
import unittest
import main

class TestExample(unittest.TestCase):

    def test_subtraction(self):
        self.assertEqual(subtraction(5, 2), 3)

if __name__ == '__main__':
    unittest.main()

The above test is verifying that the subtraction function works properly. The output of the test function is a Boolean.
