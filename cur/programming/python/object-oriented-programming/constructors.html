Recall the code thus far for our book class: 


class Book:
    language = 'English'
    worn_down = False

In addition to having attributes like above, we can also include functions within our class. We call these functions methods; these methods are specific to our class and can only be used by instances of our class. There is a special method which we use to initialize, or create, objects, known as the __init__ method (two underscores on each side). This method is called a constructor and is the key to initializing objects with instance attributes. Keep in mind that this method is very useful to include for all classes, but it is not required.

Here is what our code would look like if we wanted to initialize an object with certain attributes : 


class Book:
        	language = 'English'
        	worn_down = False

        	def __init__(self, genre, title, author, publication_year):
        	     self.genre = genre
        	     self.title = title
        	     self.author = author
        	     self.publication_year = publication_year

Now, when we create a Book object, we must pass in the proper number of arguments, which in this case is 4:

>>> hunger_games = Book('Dystopian fiction', 'The Hunger Games', 'Suzanne Collins', 2008)
>>> hunger_games.genre
 'Dystopian fiction'
>>> hunger_games.publication_year
 2008
>>> hunger_games.language
 'English'


Key things to note:

  1. If you are wondering why we did not pass anything in for the argument self, the answer is that we did. Self is automatically bound to whatever name/variable we bind our object to, which in this case is hunger_games. Inside our __init__ function, we have to type "self.[insert attribute here]" on the left side of the equal sign because this uses dot notation to bind the instance attribute we pass in to whatever name/variable we put on the right side of the dot. Without the self, our code would not error right away, but Python wouldnt recognize what we typed as instance attribute, so it would error when we tried to later ask for those instance attributes.
  2. We must ensure that any arguments we pass into __init__ that are words are properly quoted, so that Python can recognize them as string types.
  3. Even though we never specifically passed in a language when we created our hunger_games object, we can still access its language (when typing in hunger_games.language) because earlier we coded a class attribute language shared by ALL book instances.
